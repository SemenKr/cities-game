# Игра в города на время

Интерактивная браузерная игра на React, где пользователь соревнуется с компьютером в классические "Города". Проект сделан как frontend pet-project с упором на игровой UX, адаптивный интерфейс, явную игровую логику и тестируемость.

## Demo

- Live demo: [semenkr.github.io/cities-game](https://semenkr.github.io/cities-game/)
- Repository: [github.com/SemenKr/cities-game](https://github.com/SemenKr/cities-game)

## Что умеет приложение

- игра против компьютера с проверкой правил и повторов;
- нормализация ввода: регистр, пробелы, `е/ё`;
- выбор длительности таймера перед стартом;
- три уровня сложности компьютера;
- подсказки по допустимым городам;
- причины завершения партии на финальном экране;
- накопительная статистика игрока в `localStorage`;
- адаптивный интерфейс для desktop и mobile;
- unit- и scenario-тесты на ключевые игровые сценарии.

## Основные продуктовые решения

### 1. Игровая логика отделена от UI

Правила игры вынесены в отдельные модули, а не размазаны по JSX:

- [`src/lib/cityRules.ts`](./src/lib/cityRules.ts)
- [`src/lib/gameStats.ts`](./src/lib/gameStats.ts)

Это упростило поддержку и позволило покрыть критичное поведение тестами.

### 2. У компьютера есть уровни сложности

Компьютер не просто берет первый подходящий город:

- `easy` оставляет игроку больше возможных ответов;
- `medium` выбирает сбалансированный ход;
- `hard` старается минимизировать число ответов игрока.

### 3. Игра объясняет результат раунда

Финальный экран показывает не только победу или поражение, но и конкретную причину завершения:

- закончилось время у игрока;
- закончилось время у компьютера;
- у компьютера не осталось подходящих городов.

### 4. Состояние игрока сохраняется между раундами

На стартовом и финальном экранах отображаются:

- общее число партий;
- победы и поражения;
- лучший раунд;
- среднее число городов за игру;
- текущая серия.

## Тесты

Проект покрыт двумя уровнями проверок:

- unit-тесты для игровых правил и статистики;
- scenario-тесты для пользовательского потока игры.

Проверяются, в частности:

- выбор хода компьютера на разных уровнях сложности;
- проигрыш по таймеру;
- победа, если у компьютера нет ответа;
- обновление статистики после завершения партии.
